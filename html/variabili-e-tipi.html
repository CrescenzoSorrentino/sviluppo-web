JavaScript: Variabili e Tipi di Dati

Le variabili sono uno dei concetti fondamentali in JavaScript e in qualsiasi linguaggio di programmazione. Rappresentano contenitori per memorizzare dati che possono essere utilizzati e manipolati all'interno del codice. In questa guida completa, esploreremo come dichiarare e utilizzare variabili in JavaScript, i diversi tipi di dati disponibili e le best practices da seguire.

Dichiarazione delle variabili in JavaScript

In JavaScript moderno (ES6+), esistono tre modi principali per dichiarare variabili:

1. let - Variabili con scope di blocco

let è il modo consigliato per dichiarare variabili che cambieranno valore durante l'esecuzione del programma. Le variabili dichiarate con let hanno uno scope limitato al blocco in cui sono definite.

let nome = "Mario";
let età = 30;
let isAttivo = true;

// Le variabili let possono essere riassegnate
nome = "Luigi";

2. const - Costanti con scope di blocco

const viene utilizzato per dichiarare variabili il cui valore non cambierà dopo l'assegnazione iniziale. Come let, anche const ha uno scope di blocco.

const PI = 3.14159;
const NOME_APP = "BitCafé";

// Questo genererà un errore
// PI = 3.14; // TypeError: Assignment to constant variable

Nota importante: Quando si dichiara un oggetto o un array con const, il contenuto dell'oggetto o dell'array può essere modificato, ma non è possibile riassegnare la variabile stessa.

const persona = { nome: "Mario", età: 30 };
persona.nome = "Luigi"; // Questo è valido
// persona = { nome: "Luigi", età: 30 }; // Questo genererà un errore

3. var - Variabili con scope di funzione (legacy)

var è il modo più vecchio per dichiarare variabili in JavaScript. A differenza di let e const, le variabili dichiarate con var hanno uno scope di funzione anziché di blocco, e possono essere "hoisted" (sollevate) all'inizio della funzione.

var messaggio = "Ciao mondo";

function esempio() {
  var x = 10; // scope limitato alla funzione
  if (true) {
    var y = 20; // ancora visibile fuori dal blocco if
    let z = 30; // scope limitato al blocco if
  }
  console.log(x); // 10
  console.log(y); // 20
  console.log(z); // ReferenceError: z is not defined
}

Nei progetti moderni, è consigliabile utilizzare let e const invece di var per evitare comportamenti inaspettati legati allo scope e all'hoisting.

Consiglio per lo sviluppo

Usa const per default e passa a let solo quando sai che il valore della variabile cambierà. Questo rende il codice più prevedibile e facilita l'individuazione di bug.

Tipi di dati primitivi in JavaScript

JavaScript ha sette tipi di dati primitivi:

1. String (Stringa)

Le stringhe rappresentano testo e sono racchiuse tra apici singoli, doppi o backtick.

let nome = "Mario";
let cognome = 'Rossi';
let saluto = `Ciao, ${nome} ${cognome}`; // Template string (ES6)

Le template string (introdotte in ES6) permettono di incorporare espressioni JavaScript all'interno di stringhe usando la sintassi ${espressione}.

2. Number (Numero)

A differenza di altri linguaggi, JavaScript ha un solo tipo numerico che rappresenta sia interi che numeri a virgola mobile.

let intero = 42;
let decimale = 3.14;
let esponenziale = 2.998e8; // 2.998 × 10^8
let infinito = Infinity;
let nonNumero = NaN; // Not a Number

3. Boolean (Booleano)

I booleani rappresentano valori di verità e possono essere solo true o false.

let isAttivo = true;
let haCompletato = false;

4. Undefined

undefined rappresenta una variabile che è stata dichiarata ma non ha ancora un valore assegnato.

let variabile;
console.log(variabile); // undefined

5. Null

null rappresenta l'assenza intenzionale di qualsiasi valore o oggetto.

let utente = null; // l'utente non è ancora stato caricato

6. Symbol (ES6)

I Symbol sono un tipo di dato unico e immutabile, spesso utilizzato come identificatori per proprietà di oggetti.

const id = Symbol('id');
const oggetto = {
  [id]: 123
};

7. BigInt (ES2020)

BigInt è un tipo numerico che può rappresentare interi di lunghezza arbitraria.

const numeroGrande = 9007199254740991n; // la 'n' alla fine indica un BigInt
Verifica del tipo

Puoi utilizzare l'operatore typeof per verificare il tipo di una variabile:

console.log(typeof "Ciao"); // "string"
console.log(typeof 42); // "number"
console.log(typeof true); // "boolean"

Tipi di dati complessi (riferimento)

Oltre ai tipi primitivi, JavaScript ha tipi di dati complessi che sono passati per riferimento anziché per valore.

1. Object (Oggetto)

Gli oggetti sono collezioni di coppie chiave-valore e sono fondamentali in JavaScript.

const persona = {
  nome: "Mario",
  cognome: "Rossi",
  età: 30,
  indirizzo: {
    via: "Via Roma",
    città: "Milano"
  },
  saluta: function() {
    return `Ciao, sono ${this.nome} ${this.cognome}`;
  }
};

Puoi accedere alle proprietà di un oggetto usando la notazione punto o la notazione parentesi quadre:

console.log(persona.nome); // "Mario"
console.log(persona["cognome"]); // "Rossi"
console.log(persona.saluta()); // "Ciao, sono Mario Rossi"

2. Array

Gli array sono oggetti speciali utilizzati per memorizzare sequenze ordinate di valori.

const numeri = [1, 2, 3, 4, 5];
const misto = [1, "due", true, { chiave: "valore" }, [6, 7, 8]];

console.log(numeri[0]); // 1
console.log(misto[3].chiave); // "valore"

Gli array in JavaScript hanno molti metodi utili per la manipolazione dei dati:

numeri.push(6); // Aggiunge un elemento alla fine
numeri.pop(); // Rimuove l'ultimo elemento
numeri.unshift(0); // Aggiunge un elemento all'inizio
numeri.shift(); // Rimuove il primo elemento

// Metodi funzionali (ES6+)
const doppi = numeri.map(n => n * 2); // [2, 4, 6, 8, 10]
const pari = numeri.filter(n => n % 2 === 0); // [2, 4]
const somma = numeri.reduce((acc, n) => acc + n, 0); // 15

3. Function (Funzione)

In JavaScript, le funzioni sono oggetti di prima classe e possono essere assegnate a variabili, passate come argomenti e restituite da altre funzioni.

// Dichiarazione di funzione
function somma(a, b) {
  return a + b;
}

// Espressione di funzione
const moltiplica = function(a, b) {
  return a * b;
};

// Arrow function (ES6)
const dividi = (a, b) => a / b;
Differenza tra tipi primitivi e di riferimento

I tipi primitivi sono passati per valore, mentre i tipi complessi sono passati per riferimento:

// Primitivi (passati per valore)
let a = 5;
let b = a;
a = 10;
console.log(b); // 5 (non cambia)

// Oggetti (passati per riferimento)
let obj1 = { valore: 5 };
let obj2 = obj1;
obj1.valore = 10;
console.log(obj2.valore); // 10 (cambia)

Type Coercion e Conversione dei Tipi

JavaScript è un linguaggio a tipizzazione dinamica, il che significa che le variabili possono cambiare tipo durante l'esecuzione. Inoltre, JavaScript esegue automaticamente la conversione dei tipi (coercion) in determinate operazioni.

Coercion implicita

console.log("5" + 2); // "52" (il numero viene convertito in stringa)
console.log("5" - 2); // 3 (la stringa viene convertita in numero)
console.log("5" * "2"); // 10 (entrambe le stringhe vengono convertite in numeri)
console.log(5 + true); // 6 (true viene convertito in 1)
console.log(5 + false); // 5 (false viene convertito in 0)

Conversione esplicita

È possibile convertire esplicitamente i tipi utilizzando funzioni come String(), Number(), Boolean(), ecc.

// Conversione in stringa
console.log(String(42)); // "42"
console.log(String(true)); // "true"
console.log(42 + ""); // "42"

// Conversione in numero
console.log(Number("42")); // 42
console.log(Number("42px")); // NaN
console.log(Number(true)); // 1
console.log(Number(false)); // 0
console.log(+"42"); // 42 (l'operatore unario + converte in numero)

// Conversione in booleano
console.log(Boolean(42)); // true
console.log(Boolean(0)); // false
console.log(Boolean("")); // false
console.log(Boolean("Ciao")); // true
console.log(Boolean(null)); // false
console.log(Boolean(undefined)); // false
Attenzione

La coercion implicita può portare a comportamenti inaspettati. Per confronti più sicuri, usa sempre l'operatore di uguaglianza stretta (===) anziché l'uguaglianza semplice (==).

console.log("5" == 5); // true (con coercion)
console.log("5" === 5); // false (senza coercion)

Best Practices per l'uso di Variabili e Tipi in JavaScript

  1. Usa const per default, passa a let solo quando necessario.
  2. Evita var nei progetti moderni.
  3. Usa nomi di variabili descrittivi che spiegano chiaramente lo scopo della variabile.
  4. Segui le convenzioni di naming: camelCase per variabili e funzioni, PascalCase per classi.
  5. Usa l'operatore di uguaglianza stretta (=== e !==) per evitare problemi di coercion.
  6. Evita variabili globali quando possibile.
  7. Inizializza sempre le variabili al momento della dichiarazione.
  8. Usa le template string per concatenare stringhe in modo più leggibile.
