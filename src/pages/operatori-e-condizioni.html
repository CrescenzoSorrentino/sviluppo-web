JavaScript: Operatori e Strutture Condizionali

Gli operatori e le strutture condizionali sono elementi fondamentali di JavaScript che permettono di manipolare dati e controllare il flusso di esecuzione del codice. In questa guida completa, esploreremo i diversi tipi di operatori disponibili in JavaScript e come utilizzare le strutture condizionali per creare logica decisionale nei tuoi programmi.

Operatori aritmetici in JavaScript

Gli operatori aritmetici eseguono operazioni matematiche su numeri, come l'addizione, la sottrazione, la moltiplicazione e la divisione.

Operatore Descrizione Esempio Risultato
+ Addizione 5 + 2 7
- Sottrazione 5 - 2 3
* Moltiplicazione 5 * 2 10
/ Divisione 5 / 2 2.5
% Modulo (resto) 5 % 2 1
** Esponente (ES2016) 5 ** 2 25
++ Incremento let x = 5; x++; x diventa 6
-- Decremento let x = 5; x--; x diventa 4

Esempi di operatori aritmetici

// Operazioni di base
let somma = 10 + 5;      // 15
let differenza = 10 - 5;  // 5
let prodotto = 10 * 5;    // 50
let quoziente = 10 / 5;   // 2
let resto = 10 % 3;       // 1 (il resto di 10 diviso 3)
let potenza = 10 ** 2;    // 100 (10 elevato alla 2)

// Incremento e decremento
let contatore = 0;
contatore++;              // contatore è ora 1 (post-incremento)
++contatore;              // contatore è ora 2 (pre-incremento)
contatore--;              // contatore è ora 1 (post-decremento)
--contatore;              // contatore è ora 0 (pre-decremento)
Differenza tra pre e post incremento/decremento

Con il pre-incremento (++x), la variabile viene incrementata prima di essere utilizzata nell'espressione. Con il post-incremento (x++), la variabile viene incrementata dopo essere stata utilizzata nell'espressione.

let a = 5;
let b = ++a;  // a viene incrementato a 6, poi b diventa 6
console.log(a, b);  // 6, 6

let c = 5;
let d = c++;  // d diventa 5, poi c viene incrementato a 6
console.log(c, d);  // 6, 5

Operatori di assegnazione

Gli operatori di assegnazione assegnano valori alle variabili JavaScript. L'operatore di assegnazione base è =, ma esistono anche operatori composti che combinano un'operazione aritmetica con l'assegnazione.

Operatore Descrizione Esempio Equivalente a
= Assegnazione x = y x = y
+= Addizione e assegnazione x += y x = x + y
-= Sottrazione e assegnazione x -= y x = x - y
*= Moltiplicazione e assegnazione x *= y x = x * y
/= Divisione e assegnazione x /= y x = x / y
%= Modulo e assegnazione x %= y x = x % y
**= Esponente e assegnazione x **= y x = x ** y

Esempi di operatori di assegnazione

let x = 10;
x += 5;  // x = x + 5, ora x è 15
x -= 3;  // x = x - 3, ora x è 12
x *= 2;  // x = x * 2, ora x è 24
x /= 4;  // x = x / 4, ora x è 6
x %= 4;  // x = x % 4, ora x è 2
x **= 3; // x = x ** 3, ora x è 8 (2 elevato alla 3)

Operatori di confronto

Gli operatori di confronto confrontano i loro operandi e restituiscono un valore booleano (true o false) in base al risultato del confronto.

Operatore Descrizione Esempio Risultato
== Uguale (con coercion) 5 == "5" true
=== Strettamente uguale (senza coercion) 5 === "5" false
!= Diverso (con coercion) 5 != "6" true
!== Strettamente diverso (senza coercion) 5 !== "5" true
> Maggiore di 5 > 3 true
< Minore di 5 < 3 false
>= Maggiore o uguale a 5 >= 5 true
<= Minore o uguale a 5 <= 3 false

Uguaglianza stretta vs uguaglianza semplice

È importante comprendere la differenza tra gli operatori di uguaglianza stretta (===, !==) e gli operatori di uguaglianza semplice (==, !=):

// Uguaglianza semplice (con coercion)
console.log(5 == "5");     // true (il tipo viene convertito)
console.log(0 == false);   // true (il tipo viene convertito)
console.log(null == undefined); // true

// Uguaglianza stretta (senza coercion)
console.log(5 === "5");    // false (tipi diversi)
console.log(0 === false);  // false (tipi diversi)
console.log(null === undefined); // false
Best Practice

È generalmente consigliato utilizzare gli operatori di uguaglianza stretta (=== e !==) per evitare comportamenti inaspettati dovuti alla coercion dei tipi.

Operatori logici

Gli operatori logici sono utilizzati per determinare la logica tra variabili o valori e restituiscono un valore booleano.

Operatore Descrizione Esempio Risultato
&& AND logico true && true true
|| OR logico true || false true
! NOT logico !true false

Esempi di operatori logici

// AND logico (&&)
console.log(true && true);   // true
console.log(true && false);  // false
console.log(false && true);  // false
console.log(false && false); // false

// OR logico (||)
console.log(true || true);   // true
console.log(true || false);  // true
console.log(false || true);  // true
console.log(false || false); // false

// NOT logico (!)
console.log(!true);  // false
console.log(!false); // true

Short-circuit evaluation (valutazione a corto circuito)

Gli operatori logici && e || utilizzano la valutazione a corto circuito, il che significa che il secondo operando viene valutato solo se necessario:

// AND (&&): se il primo operando è falsy, restituisce il primo operando senza valutare il secondo
console.log(false && qualcosaCheCausaErrore); // false (il secondo operando non viene valutato)

// OR (||): se il primo operando è truthy, restituisce il primo operando senza valutare il secondo
console.log(true || qualcosaCheCausaErrore); // true (il secondo operando non viene valutato)

// Utilizzo comune: impostare valori di default
const nome = inputUtente || "Utente anonimo";

// Utilizzo comune: eseguire una funzione solo se una condizione è vera
isAdmin && eseguiFunzioneAdmin();

Operatore di coalescenza nulla (??)

L'operatore di coalescenza nulla (??) è un operatore logico che restituisce il suo operando di destra quando l'operando di sinistra è null o undefined, altrimenti restituisce l'operando di sinistra.

const nome = null;
const nomeVisualizzato = nome ?? "Utente anonimo"; // "Utente anonimo"

const età = 0;
const etàVisualizzata = età ?? 18; // 0 (poiché 0 non è null o undefined)
Differenza tra ?? e ||

L'operatore ?? considera solo null e undefined come valori falsy, mentre || considera anche 0, "", false, NaN come falsy.

Strutture condizionali if-else

Le strutture condizionali if-else permettono di eseguire blocchi di codice diversi in base a condizioni specifiche.

Sintassi di base if-else

// Struttura if semplice
if (condizione) {
  // Codice eseguito se la condizione è vera
}

// Struttura if-else
if (condizione) {
  // Codice eseguito se la condizione è vera
} else {
  // Codice eseguito se la condizione è falsa
}

// Struttura if-else if-else
if (condizione1) {
  // Codice eseguito se condizione1 è vera
} else if (condizione2) {
  // Codice eseguito se condizione1 è falsa e condizione2 è vera
} else {
  // Codice eseguito se entrambe le condizioni sono false
}

Esempi pratici

// Esempio 1: Controllo dell'età
const età = 18;

if (età >= 18) {
  console.log("Sei maggiorenne");
} else {
  console.log("Sei minorenne");
}

// Esempio 2: Valutazione di un voto
const voto = 85;

if (voto >= 90) {
  console.log("Eccellente");
} else if (voto >= 80) {
  console.log("Molto buono");
} else if (voto >= 70) {
  console.log("Buono");
} else if (voto >= 60) {
  console.log("Sufficiente");
} else {
  console.log("Insufficiente");
}

Condizioni multiple

Puoi combinare più condizioni utilizzando gli operatori logici:

// Esempio: Controllo di accesso
const isAdmin = true;
const isLoggedIn = true;
const hasPermission = false;

if (isLoggedIn && (isAdmin || hasPermission)) {
  console.log("Accesso consentito");
} else {
  console.log("Accesso negato");
}

Istruzione switch

L'istruzione switch è un'alternativa a più istruzioni if-else quando si vuole confrontare una variabile con più valori possibili.

Sintassi di base

switch (espressione) {
  case valore1:
    // Codice eseguito se espressione === valore1
    break;
  case valore2:
    // Codice eseguito se espressione === valore2
    break;
  // Altri casi...
  default:
    // Codice eseguito se nessun caso corrisponde
}
Importante: l'istruzione break

L'istruzione break è fondamentale in uno switch. Senza di essa, l'esecuzione continuerebbe nei casi successivi anche dopo aver trovato una corrispondenza (comportamento noto come "fall-through").

Esempio di switch

const giorno = new Date().getDay(); // 0-6 (0 è Domenica)

switch (giorno) {
  case 0:
    console.log("Domenica");
    break;
  case 1:
    console.log("Lunedì");
    break;
  case 2:
    console.log("Martedì");
    break;
  case 3:
    console.log("Mercoledì");
    break;
  case 4:
    console.log("Giovedì");
    break;
  case 5:
    console.log("Venerdì");
    break;
  case 6:
    console.log("Sabato");
    break;
  default:
    console.log("Giorno non valido");
}

Fall-through intenzionale

A volte, il comportamento di fall-through può essere utile quando si vuole che più casi eseguano lo stesso codice:

const mese = new Date().getMonth(); // 0-11

switch (mese) {
  case 11:
  case 0:
  case 1:
    console.log("Inverno");
    break;
  case 2:
  case 3:
  case 4:
    console.log("Primavera");
    break;
  case 5:
  case 6:
  case 7:
    console.log("Estate");
    break;
  case 8:
  case 9:
  case 10:
    console.log("Autunno");
    break;
  default:
    console.log("Mese non valido");
}

Operatore ternario

L'operatore ternario (? :) è un'espressione condizionale concisa che può sostituire semplici istruzioni if-else.

Sintassi

condizione ? espressioneSeVera : espressioneSeFalsa

Esempi di operatore ternario

// Esempio 1: Assegnazione condizionale
const età = 20;
const stato = età >= 18 ? "maggiorenne" : "minorenne";
console.log(stato); // "maggiorenne"

// Esempio 2: Restituzione condizionale in una funzione
function saluta(nome, ora) {
  return ora < 12 ? `Buongiorno, ${nome}!` : `Buonasera, ${nome}!`;
}
console.log(saluta("Mario", 10)); // "Buongiorno, Mario!"

// Esempio 3: Operatori ternari annidati (da usare con cautela)
const voto = 85;
const risultato = voto >= 90 ? "A" : voto >= 80 ? "B" : voto >= 70 ? "C" : voto >= 60 ? "D" : "F";
console.log(risultato); // "B"
Best Practice

L'operatore ternario è ottimo per assegnazioni condizionali semplici, ma per logica più complessa è preferibile utilizzare if-else per mantenere il codice leggibile. Evita di annidare troppi operatori ternari.

Best Practices per Operatori e Condizioni

  1. Usa l'uguaglianza stretta (=== e !==) invece dell'uguaglianza semplice per evitare problemi di coercion dei tipi.
  2. Mantieni le condizioni semplici. Se una condizione diventa troppo complessa, considera di suddividerla in variabili con nomi significativi.
  3. Usa le parentesi per chiarire l'ordine di valutazione nelle espressioni complesse.
  4. Sfrutta la valutazione a corto circuito degli operatori logici per scrivere codice più conciso.
  5. Usa l'operatore ternario per assegnazioni condizionali semplici, ma preferisci if-else per logica più complessa.
  6. Non dimenticare il break nelle istruzioni switch, a meno che il fall-through non sia intenzionale.
  7. Considera l'operatore di coalescenza nulla (??) per valori di default quando lavori con null o undefined.
